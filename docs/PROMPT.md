# Build Prompt

## Purpose

This document contains the implementation prompt that should be used for the next build phase of Portfolio X after documentation approval.

It is intended to preserve product direction, design constraints, content priorities, and technical boundaries so future implementation stays consistent.

---

## Implementation Prompt

Build a premium one-page portfolio website for **Akash Karthik P** named **Portfolio X**.

The website must immediately communicate that Akash is:

- Enterprise Data Engineer
- Azure Data Engineer
- ETL Modernization Specialist
- Cloud Migration Engineer
- Future Cloud Data Architect

The website should feel like a premium enterprise SaaS landing page.

It must not feel flashy, experimental, neon, cyberpunk, gaming-inspired, or terminal-heavy.

The visual quality should come from typography, spacing, clean hierarchy, restrained motion, and thoughtful structure.

---

## Tech Constraints

Use only:

- HTML5
- CSS3
- Vanilla JavaScript

Do not use:

- React
- Vue
- Angular
- Bootstrap
- Tailwind
- Any heavy framework

---

## Quality Targets

The site must be:

- SEO-friendly

- Performance
- Accessibility
- Best Practices
- SEO

## Motion Guidelines

Use only minimal motion:

- Fade
- Slide
- Hover lift
- Smooth scroll

Avoid excessive animation.

Motion should support polish and clarity, not draw attention to itself.

Respect reduced-motion accessibility preferences.

## Accessibility Requirements

Implementation must include:

- Semantic HTML5
- Proper heading order
- Keyboard-accessible navigation
- Visible focus states
- Sufficient color contrast
- Clear link and button labels
- Mobile-friendly tap targets

Do not rely on color alone to convey meaning.

---
